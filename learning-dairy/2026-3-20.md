# <span id='pointer'>2.指针</span>
## <span id='location'>地址</span>：
在C语言里，每个变量都要占据一定的内存。即内存用来存储变量。<br>内存是由一定的字节组成的( 这里占用的字节大小取决于变量类型 )。地址就是该内存的标识符，取第一个字节的16进制码作为地址。<br>以整型int为例，int占4字节，在地址里用第一个字节的地址代表该int变量的地址，***访问地址即可得到变量的值***。
- ***指针***：和”整数对应int类型“、”浮点数对应double类型“、”布尔值对应bool类型“等一样，***地址也需要它自己的类型，这就是指针类型***。如果我们说：int是存放整数的类型。那么，***指针就是存放地址的类型。指针类型也是一种变量，一种储存地址的变量。***
- 取址与解址：既然每个变量都占用内存，那么它们就都有地址，我们应该怎么取址呢？又该怎么声明指针变量呢？
<br>
## <span id='makeloc'>取址</span>
1. 我们可以利用&取址符来获取变量的地址。取址只能对**左值**(有地址的量)进行操作，无法对**右值**(常量和临时量)进行操作。<br>
2. 我们可以通过　[类型]* 变量名 = 某地址；的方法来声明和定义一个指针变量。示例如下：
```c++
int a = 1;
int* p = &a;
// like this
```
## <span id='getloc'>解址</span>
我们可以利用 \* 解址符来做到两件事：
<br>
### 重要！！！
**1. 访问地址对应的内容（即该内存地址中存放的内容）<br>
1. 更改该地址对应的内容，并且更改值和地址绑定**！！！<br>

---
## 最重要的一点！！<br>以后我们谈论指针的时候，不要想指针指向了谁。*要想指针存放了谁的地址！！！*<br><br>指针本身也是有地址的，但是在考虑它存放的地址的时候，和它本身的地址完全没有任何关系！！
- <span id='type'>**指针的类型**</span>：需要注意的是，指针变量也需要声明类型。我们可以理解为，在声明指针的时候，\*是用来声明这是指针变量的，而前面的部分**是用来声明这个指针存储的地址其存储的值是什么类型的**。例如`int** p;`这里最好把它理解为`(int*)*p`，也就是“一个指针，它存放的是`int*`类型的地址”(则它是一种二级指针)。
- <span id='addable'>**指针语法糖**</span>：那么指针类型有什么用？下面我们就要介绍一种指针语法糖。目前我学习到的指针实质上都和数组有关，所以我仅拿数组举例。<br>因为我们知道数组里的元素是是连续占用内存的，所以下标访问的方法其实很自然：给定一个起始地址（起始字节），再给定一个单位元素的字节大小（指针类型），这就是一个等差数列。这就是"**指针的语法糖**"，参考下面的式子。事实上，在数组和字符串里都有这样的下标访问法。<br>
那么起始地址是怎么给出的呢？事实上很简单，**数组的名字会退化成起始地址**，也就是数组第一个元素的指针。所以`arr[n] == *(arr + n)`这里编译器会自动帮你把后面的n乘上指针类型的字节数。<br>
### 注意数组的语法糖，这里arr1[i]是直接帮助我们干了两个事：
#### 1. 把地址往后偏移到第i+1个元素
#### 2. 自动解析地址，取出里面的数据或者对里面的数据进行操作

## <span id='new'>new：</span>**在堆上占用**<br>
new表示在"堆"上占用内存新建一些东西，可以是各种类型。这里我们拿数组举例。<br>
用new新建数组和声明数组很像，实际上就是在堆上声明数组，但是有一点不同，实质上是C语言的历史遗留问题。
```c++
//示例 1
int** arr = new int*[3];
arr[0] = new int[2];
arr[0][0] = 1;
//示例 2
arr[1] = new int(2); 
/*
一点说明
我们能看到在new里声明数组的时候是int[n], 但是原生数组声明是 int arr[n];实际上按理来说应该是int[n] arr;不过无所谓了，也无法改变
*/

```
我们一点一点来拆解一下：首先看示例一<br><br>
**第一行**：这里new新建的东西就是 3个int*：3个存放int地址的指针，注意这三个指针本身的地址是连续的( 且占8字节 )。**重点：new会返回新建的东西的地址**。这里新建了三个int*，就返回第一个int*的地址。<br>
返回的地址被赋值给左边的`int** arr`二级指针。arr本身是指针，现在里面存放的就是首个`int*`的地址，且3个`int*`地址是连续的。<br><br>
**第二行**：`arr[0]`, 访问arr存放的地址，我们访问到了一个地址（指针变量），并且可以对其进行修改，让它变成`new int[2]`返回的地址。<br>

**第三行**：`arr[0][0] = 1`, 访问`arr[0]`存放的地址（一个int指针）里存放的地址，访问到了一个整数，并且把它修改为1。<br>

再看示例二<br>
先访问arr后面紧挨着的地址，访问到一个地址（int指针）, 然后把它修改成另外一个地址，这个地址指向整数2。

# <span id='string'>3. 字符串</span>
终于到字符串了。
```c++
#include <bits/stdc++.h>
// 这里面包含了<string>
using namespace std;

string s1 = "Hello world!";
// 下面每个新模块的s1都假设保持这个样子
string s2 = "12345"
```
这样就声明了一个字符串<br>
**直接进入重点语法**
```c++
// 1.输入输出
getline(cin,s); //读取整行内容，丢弃回车
cout << s << endl;

// 2.增删改查
// 增：拼接
s1 = "Yo," + s1 + s2 + "666";// Yo,Hello world!12345666
// 增：插入
int pos = 5;
s1.insert(pos,s2); // inplace原地修改，有返回值，返回修改后自己的引用
//pos表示插入的位置，s2表示要插入的字符串（pos是插入后s2的首个字符的下标！！！）
// s1 = "Hello12345 world!"

// 删
int len = 6;
s1.erase(pos,len);
// 从下标为pos的开始，删除长度为len的内容，pos不能越界，len无所谓。
// s1 = "Hello!" 删掉了“ world”

// 改
s1[1] = 'a'; // 直接下标修改，注意右边必须是char类型，单引号
s1.replace(pos,len,s2,1,3);
// 这里用到了replace替换，仍然是下标+长度，中间是要替换的字符串，右边也是下标+长度
// s1 = "Hallo234!"

// 查
// 下标法查直接略过
string s3 = "al";
s.find(s3,pos);
//从pos下标开始，找到最近的s3子串
// 重点！！！！！如果找不到会返回string::npos!!!

// 3. 格式判断
// 下面这些函数不是string库里的
char x = '1' // 这里储存的是ASCII码
isdigit(x); // 判断数字
isspace(x); // 判断空格、回车等
isalpha(x); // 判断字母
isupper(x); islower(x); // 判断大小写
toupper(x); tolower(x); // 转换大小写

// 4. 其他
reverse(s1.begin(),s1.end()); // 把s1整体逆序
// 实际上是这样的，reverse函数要求里面放两个迭代器(一种类内定义的指针），会把第一个指针到第二个指针内的地址对应元素逆序。s1.begin()这里就返回的是第一个元素的地址。
// 注意右边取不到，s.end()实际上是最后一个元素的下一个地址
s.substr(pos,len);
// 又遇到了pos，len组合，这是取子串，len同样可以越界
```
## <span id='thinking'>四、算法思想：双指针</span>
参考string-learning/string14v2.cpp && string-learning/string15.cpp<br>
双指针思想实际上和指针关系不大，可以称为双position思想。实质上是:<br>
固定一个pos，放出next-pos寻路，找到落脚点后固定next-pos，对pos和next-pos之间的内容进行处理，然后再把pos拉至next-pos处固定，然后放出next-pos寻路。按照这个过程循环。<br>
重点在于：<br>
1. nextpos的设定和寻路方式
2. 题目逻辑要求我们对pos和nextpos之间的内容怎么处理
3. pos拉到next-pos的方式以及是否能循环下去
4. 边界处理可以最后考虑，双指针一般来说边界条件不会很多。

<br>**这种题反而先从中间思考，把逻辑构建出来，再去考虑边界，会好做一些**
